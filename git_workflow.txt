Git Workflow for IAPP-Practice Project

1. Initialize Repository
   git init
   git add .
   git commit -m "Initial commit"

2. Daily Development
   git add .
   git commit -m "Added member registration functionality"
   git push origin main

3. Feature Development
   git checkout -b feature/new-feature
   git add .
   git commit -m "Implemented new feature"
   git checkout main
   git merge feature/new-feature
   git push origin main

4. Bug Fixes
   git add .
   git commit -m "Fixed database connection issue"
   git push origin main

5. Regular Commits
   - Always commit working code
   - Use descriptive commit messages
   - Push to remote repository regularly
   - Keep main branch clean and stable

6. File Management
   - Add all files: git add .
   - Commit changes: git commit -m "message"
   - Push to remote: git push origin main
   - Pull latest: git pull origin main

7. Branch Strategy
   - main: stable production code
   - Different branches: for different things

8. Common Commands
   git status
   git log --oneline
   git branch
   git log
